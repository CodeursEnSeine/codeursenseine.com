Démarrer un nouveau projet, c’est répéter les mêmes étapes : poser l’architecture, configurer la base de données, coder le backend et le frontend, écrire les tests… et recommencer encore. Aujourd’hui, les AI Coding Assistants permettent de se faire accompagner tout au long de ce processus, mais l’écosystème est dominé par des solutions propriétaires.

Dans ce talk, je partagerai un retour d’expérience concret sur l’intégration d’outils open source comme TabbyML, Continue.dev et Devoxx Genie dans un workflow de développement. Nous verrons comment ces assistants peuvent générer du code et des tests, documenter automatiquement des modules existants et automatiser certaines tâches de maintenance, le tout sans perdre la maîtrise du code produit.

Une démonstration live montrera comment passer de zéro à une application fonctionnelle, du premier commit au déploiement, tout en abordant les limites rencontrées sur le terrain, notamment la qualité, la sécurité et la dépendance. L’objectif est de montrer comment intégrer efficacement un AI Coding Assistant dans un environnement professionnel, de manière pragmatique et contrôlée.
