En 2012, les CSS atomiques font leur première apparition avec, entre autres, l’émergence de ACSS (Atomized CSS).

Sur le papier, l’idée est simple : styliser nous-mêmes nos éléments HTML à l’aide de classes utilitaires où classes atomiques.

Il s’agissait, à l’époque, d’un changement majeur dans la façon de composer nos pages. En effet, cette approche CSS s’opposait totalement à l’approche sémantique classique, par composants d’interface, introduite notamment par Bootstrap.

Je me souviens que je faisais partie des nombreux développeurs à être opposés à cette nouvelle méthodologie d’intégration, contraire, selon moi, aux bonnes pratiques CSS.

10 ans plus tard, me voilà proposer une conférence sur Tailwind CSS, le framework ayant démocratisé cette approche, que l’on nomme aujourd’hui l’approche CSS Utility-First… c’est un comble !

Dans cette conférence, j’aborderai les principes des frameworks CSS sémantiques et Utility-First en vous partageant mes retours d’expérience sur ces deux approches CSS. Puis, je ferai un tour d’horizon de Tailwind CSS et de la façon dont je l’utilise au quotidien en conciliant le meilleur des deux mondes.

Teaser : Je parlerai, peut-être, des propriétés personnalisées CSS !
