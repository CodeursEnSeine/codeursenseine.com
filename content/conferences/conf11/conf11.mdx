Imaginez-vous en plein travail sur une nouvelle fonctionnalité et vous devez absolument faire une revue de code d'un de vos collègues.
Vous allez encore une fois mettre de côté votre code en cours, récupérer celui de votre collègue et qui sait peut être modifier votre environnement local pour tester ses changements?

Et si nous vous proposions une nouvelle expérience de développement? Comment?
Et pourquoi pas par un simple Copier/Coller votre l'url de repository GIT dans Docker Desktop?

Les Dev Environments est une manière d'isoler votre code, vos dépendances et processus en cours, vous permettant ainsi d'avoir plusieurs versions d'un même projet en test sur votre machine.
Et bien plus encore, partager simplement votre code avec les autres membres de votre équipe, interagissez via Docker Compose avec une stack applicative complexe ...
