A l'heure d'aujourd'hui, l'éco-conception des applications web est un domaine qui monte. Vous pouvez faire des tests de performance de vos pages web, savoir combien d'énergie les utilisateurs auront besoin pour utiliser votre interface, etc. Mais qu'en est-il du serveur ? Quelles sont les bonnes pratiques ? Faut il pour autant écrire mes scripts de déploiement en C ? Nous allons voir quelles métriques sont bonnes a regarder, quelles sont les personnes a convaincre pour faire de l’éco-conception coté serveur et avec quels arguments.
